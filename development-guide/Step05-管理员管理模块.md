# Step 05 - 管理员管理模块

## 目标
实现管理员的完整CRUD操作，包括列表查询、新增、编辑、删除、状态管理

## 后端实现

### 5.1 管理员模块搭建
- 创建 `admin` 模块（controller、service、module）
- 创建管理员相关 DTO（CreateAdminDto、UpdateAdminDto、QueryAdminDto）
- 配置模块依赖（TypeORM、JWT 等）

### 5.2 DTO 定义

#### QueryAdminDto（查询参数）
**字段**：
- `page`：页码，默认1
- `pageSize`：每页数量，默认10
- `username`：用户名（模糊搜索）
- `nickname`：昵称（模糊搜索）
- `email`：邮箱（模糊搜索）
- `mobile`：手机号（模糊搜索）
- `group_id`：管理员组ID
- `status`：状态筛选
- `created_at_start`：创建时间开始
- `created_at_end`：创建时间结束

#### CreateAdminDto（创建管理员）
**字段**：
- `username`：用户名（必填，唯一，2-20字符）
- `nickname`：昵称（必填，2-50字符）
- `password`：密码（必填，6-20字符，需包含字母和数字）
- `email`：邮箱（必填，邮箱格式）
- `mobile`：手机号（必填，11位数字）
- `group_id`：管理员组ID（必填）
- `avatar`：头像URL（可选）
- `status`：状态（默认 normal）

**验证规则**：
- 使用 class-validator 装饰器验证
- username、email、mobile 唯一性验证
- 密码强度验证

#### UpdateAdminDto（更新管理员）
**字段**：
- 与 CreateAdminDto 类似，但都是可选字段
- 密码可选（不填则不修改）
- 不允许修改 username（或需要特殊权限）

### 5.3 管理员列表接口
**接口**：`GET /api/admins`

**权限**：`system:admin:view`

**查询参数**：参考 QueryAdminDto

**业务逻辑**：
1. 构建查询条件（模糊搜索、精确匹配、日期范围）
2. 联表查询管理员组信息（可选）
3. 分页查询
4. 排序（默认按创建时间倒序）
5. 隐藏敏感字段（password、token）
6. 返回列表数据和分页信息

**响应格式**：
```
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": 1,
        "username": "admin",
        "nickname": "超级管理员",
        "email": "admin@example.com",
        "mobile": "13800138000",
        "avatar": null,
        "group_id": 1,
        "group_name": "超级管理员组",
        "status": "normal",
        "login_at": "2024-01-01 12:00:00",
        "login_ip": "127.0.0.1",
        "created_at": "2024-01-01 00:00:00",
        "updated_at": "2024-01-01 12:00:00"
      }
    ],
    "pagination": {
      "page": 1,
      "pageSize": 10,
      "total": 100,
      "totalPages": 10
    }
  }
}
```

### 5.4 管理员详情接口
**接口**：`GET /api/admins/:id`

**权限**：`system:admin:view`

**业务逻辑**：
1. 根据 ID 查询管理员
2. 联表查询管理员组信息
3. 隐藏敏感字段
4. 管理员不存在时返回 404

### 5.5 创建管理员接口
**接口**：`POST /api/admins`

**权限**：`system:admin:create`

**请求参数**：参考 CreateAdminDto

**业务逻辑**：
1. DTO 验证（格式、必填项）
2. 检查 username 是否已存在
3. 检查 email 是否已存在
4. 检查 mobile 是否已存在
5. 验证管理员组是否存在
6. 密码使用 bcrypt 加密（轮数10）
7. 创建管理员记录
8. 记录操作日志
9. 返回创建成功的管理员信息（不含密码）

**错误处理**：
- 用户名已存在：400
- 邮箱已存在：400
- 手机号已存在：400
- 管理员组不存在：400

### 5.6 更新管理员接口
**接口**：`PUT /api/admins/:id`

**权限**：`system:admin:edit`

**请求参数**：参考 UpdateAdminDto

**业务逻辑**：
1. DTO 验证
2. 检查管理员是否存在
3. 如果修改 email，检查新邮箱是否被占用
4. 如果修改 mobile，检查新手机号是否被占用
5. 如果修改 password，使用 bcrypt 加密新密码
6. 如果修改 group_id，验证新管理员组是否存在
7. 不允许修改自己的管理员组（可选安全策略）
8. 更新管理员记录
9. 记录操作日志
10. 返回更新后的管理员信息

**特殊处理**：
- 不允许编辑超级管理员（ID=1）的某些字段（可选）
- 密码字段为空时不更新密码

### 5.7 删除管理员接口
**接口**：`DELETE /api/admins/:id`

**权限**：`system:admin:delete`

**业务逻辑**：
1. 检查管理员是否存在
2. 不允许删除自己
3. 不允许删除超级管理员（ID=1）
4. 检查是否有关联数据（可选）
5. 执行删除（软删除或硬删除）
6. 记录操作日志
7. 返回成功消息

**软删除方案**：
- 添加 `deleted_at` 字段
- 删除时设置 deleted_at = 当前时间
- 查询时自动过滤 deleted_at 不为空的记录

### 5.8 批量删除接口（可选）
**接口**：`DELETE /api/admins/batch`

**请求参数**：
- `ids`：管理员ID数组

**业务逻辑**：
- 批量验证和删除
- 不允许删除自己和超级管理员

### 5.9 修改状态接口
**接口**：`PATCH /api/admins/:id/status`

**权限**：`system:admin:edit`

**请求参数**：
- `status`：新状态（normal/hidden）

**业务逻辑**：
1. 验证状态值有效性
2. 不允许禁用自己
3. 不允许禁用超级管理员
4. 更新状态
5. 如果设置为 hidden，可考虑强制登出该管理员

### 5.10 重置密码接口
**接口**：`POST /api/admins/:id/reset-password`

**权限**：`system:admin:edit`

**请求参数**：
- `new_password`：新密码（可选，不填则生成随机密码）

**业务逻辑**：
1. 生成或接收新密码
2. 验证密码强度
3. 使用 bcrypt 加密
4. 更新密码
5. 清除该管理员的 token（强制重新登录）
6. 记录操作日志
7. 返回新密码（如果是随机生成的）

### 5.11 操作日志记录
- 在所有写操作（创建、更新、删除）后记录操作日志
- 记录内容：
  - 操作人信息
  - 操作类型（create/update/delete）
  - 操作对象（管理员ID、用户名）
  - 操作详情（修改前后的值）
  - 操作时间、IP等

## 前端实现

### 6.1 管理员管理页面
**路由**：`/system/admin`

**页面结构**：
- 搜索区域（Antd Form）
- 操作按钮区（新增管理员）
- 数据表格（Antd Table）
- 分页器（Antd Pagination）

### 6.2 搜索功能
**搜索条件**：
- 用户名（输入框）
- 昵称（输入框）
- 邮箱（输入框）
- 手机号（输入框）
- 管理员组（下拉选择器）
- 状态（下拉选择器）
- 创建时间（日期范围选择器）

**功能**：
- 点击搜索按钮触发查询
- 点击重置按钮清空条件并查询
- 搜索条件可折叠/展开（可选）

### 6.3 数据表格
**表格列**：
- 序号（自动计算：(page-1)*pageSize + index + 1）
- 头像（Image 组件显示）
- 用户名
- 昵称
- 邮箱
- 手机号
- 管理员组
- 状态（Tag 组件，normal=绿色、hidden=灰色）
- 最后登录时间
- 最后登录IP
- 创建时间
- 操作（按钮组）

**操作按钮**：
- 编辑（权限：system:admin:edit）
- 删除（权限：system:admin:delete）
- 重置密码（权限：system:admin:edit）

**表格功能**：
- 多选（Checkbox，用于批量删除）
- 排序（可选）
- 固定列（操作列固定在右侧）
- Loading 状态

### 6.4 新增管理员
**触发**：点击"新增管理员"按钮

**实现方式**：抽屉（Drawer）或对话框（Modal）

**表单字段**：
- 用户名（必填，Input）
- 昵称（必填，Input）
- 密码（必填，Password，显示强度指示器）
- 确认密码（必填，与密码一致）
- 邮箱（必填，Input）
- 手机号（必填，Input）
- 管理员组（必填，Select，从接口获取管理员组列表）
- 头像（可选，Upload，上传后显示预览）
- 状态（Radio，默认 normal）

**表单验证**：
- 用户名：2-20字符，字母数字下划线
- 昵称：2-50字符
- 密码：6-20字符，必须包含字母和数字
- 确认密码：与密码一致
- 邮箱：邮箱格式
- 手机号：11位数字

**提交流程**：
1. 表单验证通过
2. 调用创建接口
3. 显示 Loading
4. 成功后：
   - 提示成功消息
   - 关闭表单
   - 刷新列表
5. 失败后显示错误提示

### 6.5 编辑管理员
**触发**：点击表格行的"编辑"按钮

**实现方式**：抽屉或对话框

**表单字段**：
- 与新增类似，但密码非必填（不填则不修改）
- 用户名禁用（不允许修改）
- 预填充当前管理员的数据

**提交流程**：
- 与新增类似，调用更新接口

### 6.6 删除管理员
**触发**：点击表格行的"删除"按钮

**实现方式**：确认对话框（Modal.confirm）

**确认提示**：
```
确定要删除管理员 "xxx" 吗？删除后将无法恢复。
```

**提交流程**：
1. 显示确认对话框
2. 用户确认后调用删除接口
3. 成功后：
   - 提示成功消息
   - 刷新列表
4. 失败后显示错误提示

### 6.7 批量删除
**触发**：选中多个管理员后，点击"批量删除"按钮

**实现方式**：确认对话框

**确认提示**：
```
确定要删除选中的 N 个管理员吗？删除后将无法恢复。
```

**提交流程**：
- 调用批量删除接口或循环调用单个删除接口

### 6.8 重置密码
**触发**：点击表格行的"重置密码"按钮

**实现方式**：对话框

**表单字段**：
- 新密码（必填，Password，显示强度指示器）
- 确认密码（必填，与新密码一致）
- 或提供"生成随机密码"按钮

**提交流程**：
1. 验证密码
2. 调用重置密码接口
3. 成功后显示新密码（如果是随机生成的）
4. 提示用户复制密码

### 6.9 API Service
创建 `src/services/admin.service.ts`：
- `getAdminList(params)`：获取管理员列表
- `getAdminDetail(id)`：获取管理员详情
- `createAdmin(data)`：创建管理员
- `updateAdmin(id, data)`：更新管理员
- `deleteAdmin(id)`：删除管理员
- `batchDeleteAdmins(ids)`：批量删除
- `updateAdminStatus(id, status)`：修改状态
- `resetAdminPassword(id, password)`：重置密码

### 6.10 组件拆分
建议拆分组件：
- `AdminList.tsx`：主页面
- `AdminSearchForm.tsx`：搜索表单
- `AdminTable.tsx`：数据表格
- `AdminForm.tsx`：新增/编辑表单（复用）
- `ResetPasswordModal.tsx`：重置密码对话框

## 注意事项

1. **安全性**：
   - 不允许删除或禁用自己
   - 超级管理员的保护机制
   - 密码传输使用 HTTPS

2. **性能**：
   - 列表查询优化（索引、分页）
   - 避免 N+1 查询问题
   - 大批量操作时的性能考虑

3. **用户体验**：
   - 操作需要确认提示
   - 错误信息友好
   - 表单验证实时反馈

4. **数据一致性**：
   - 邮箱和手机号的唯一性验证
   - 管理员组的关联验证
