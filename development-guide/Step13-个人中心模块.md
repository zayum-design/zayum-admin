# Step 13 - 个人中心模块

## 目标
实现个人信息管理、密码修改、偏好设置等个人中心功能

## 后端实现

### 13.1 个人中心模块搭建
- 创建 `profile` 模块（controller、service、module）
- 创建个人中心相关 DTO

### 13.2 DTO 定义

#### UpdateProfileDto（更新个人信息）
**字段**：
- `nickname`：昵称（可选，2-50字符）
- `avatar`：头像URL（可选）
- `email`：邮箱（可选，邮箱格式，唯一）
- `mobile`：手机号（可选，11位数字，唯一）
- `gender`：性别（可选，male/female/unknown）
- `birthday`：生日（可选）

**验证规则**：
- 如果修改邮箱，验证邮箱格式且不被其他人占用
- 如果修改手机号，验证格式且不被其他人占用

#### ChangePasswordDto（修改密码）
**字段**：
- `old_password`：旧密码（必填）
- `new_password`：新密码（必填，6-20字符，包含字母和数字）
- `confirm_password`：确认密码（必填，与新密码一致）

**验证规则**：
- 旧密码不能为空
- 新密码强度验证
- 确认密码必须与新密码一致
- 新密码不能与旧密码相同

### 13.3 获取个人信息接口
**接口**：`GET /api/profile`

**权限**：已登录

**业务逻辑**：
1. 从 token 获取当前用户ID和类型
2. 查询用户详细信息
3. 联表查询用户组信息（如果是用户）或管理员组信息（如果是管理员）
4. 查询用户权限列表
5. 隐藏密码等敏感字段
6. 返回个人信息

**响应格式**：
```
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "超级管理员",
    "email": "admin@example.com",
    "mobile": "13800138000",
    "avatar": "http://localhost:3000/uploads/avatar.jpg",
    "gender": "male",
    "birthday": "1990-01-01",
    "group_id": 1,
    "group_name": "超级管理员组",
    "status": "normal",
    "login_at": "2024-01-01 12:00:00",
    "login_ip": "127.0.0.1",
    "created_at": "2024-01-01 00:00:00",
    "permissions": [...]
  }
}
```

### 13.4 更新个人信息接口
**接口**：`PUT /api/profile`

**权限**：已登录

**请求参数**：参考 UpdateProfileDto

**业务逻辑**：
1. 从 token 获取当前用户ID和类型
2. DTO 验证
3. 如果修改邮箱，检查新邮箱是否被占用（排除自己）
4. 如果修改手机号，检查新手机号是否被占用（排除自己）
5. 更新个人信息
6. 记录操作日志
7. 返回更新后的个人信息

**特殊处理**：
- 不允许修改 username
- 不允许修改 group_id
- 不允许修改 status

### 13.5 修改密码接口
**接口**：`POST /api/profile/change-password`

**权限**：已登录

**请求参数**：参考 ChangePasswordDto

**业务逻辑**：
1. 从 token 获取当前用户ID和类型
2. DTO 验证
3. 查询用户信息
4. 使用 bcrypt 验证旧密码是否正确
5. 验证新密码不能与旧密码相同
6. 使用 bcrypt 加密新密码
7. 更新密码
8. 清除用户的所有 token（强制重新登录，可选）
9. 记录操作日志
10. 返回成功消息

**响应格式**：
```
{
  "code": 200,
  "message": "密码修改成功，请重新登录"
}
```

**错误处理**：
- 旧密码错误：400
- 新密码与旧密码相同：400
- 确认密码不一致：400

### 13.6 上传头像接口（复用）
- 复用 Step 11 的文件上传接口
- category 设置为 'avatar'
- 上传成功后返回头像URL
- 前端调用更新个人信息接口更新 avatar 字段

### 13.7 获取登录日志接口
**接口**：`GET /api/profile/login-logs`

**权限**：已登录

**查询参数**：
- `page`：页码，默认1
- `pageSize`：每页数量，默认10

**业务逻辑**：
1. 从 token 获取当前用户ID和类型
2. 查询该用户的登录日志
3. 分页查询
4. 按时间倒序排序
5. 返回登录日志列表

**用途**：
- 用户查看自己的登录历史
- 安全审计

### 13.8 获取操作日志接口
**接口**：`GET /api/profile/operation-logs`

**权限**：已登录

**查询参数**：
- `page`：页码，默认1
- `pageSize`：每页数量，默认10

**业务逻辑**：
1. 从 token 获取当前用户ID和类型
2. 查询该用户的操作日志
3. 分页查询
4. 按时间倒序排序
5. 返回操作日志列表

### 13.9 偏好设置相关接口（可选）

#### 获取用户偏好设置
**接口**：`GET /api/profile/preferences`

**业务逻辑**：
1. 查询用户偏好设置（可以是独立表或 JSON 字段）
2. 返回偏好设置

**偏好设置示例**：
```
{
  "theme": "light",           // 主题：light/dark
  "language": "zh-CN",        // 语言：zh-CN/en-US
  "sidebar_collapsed": false, // 侧边栏是否折叠
  "table_page_size": 10,      // 表格每页显示数量
  "notification_enabled": true // 是否接收通知
}
```

#### 更新用户偏好设置
**接口**：`PUT /api/profile/preferences`

**请求参数**：偏好设置对象

**业务逻辑**：
1. 验证偏好设置数据
2. 更新用户偏好设置
3. 返回更新后的偏好设置

### 14.8 主题切换实现（可选）
**浅色主题**：
- 使用 Antd 默认主题
- 或自定义浅色主题

**深色主题**：
- 使用 Antd 的 ConfigProvider 配置深色主题
- 或使用 CSS 变量切换

**切换方法**：
```typescript
import { ConfigProvider, theme } from 'antd';

<ConfigProvider
  theme={{
    algorithm: isDark ? theme.darkAlgorithm : theme.defaultAlgorithm
  }}
>
  <App />
</ConfigProvider>
```

## 注意事项

1. **安全性**：
   - 修改密码必须验证旧密码
   - 密码传输使用 HTTPS
   - 修改密码后考虑强制重新登录

2. **数据验证**：
   - 邮箱和手机号的唯一性验证（排除自己）
   - 密码强度验证
   - 表单验证及时反馈

3. **用户体验**：
   - 头像上传支持预览和裁剪
   - 密码强度可视化
   - 操作成功有明确提示

4. **隐私保护**：
   - 用户只能查看自己的日志
   - 敏感信息不展示（如完整手机号可脱敏）

5. **偏好设置**：
   - 偏好设置实时生效
   - 偏好设置持久化
   - 提供重置默认功能

6. **国际化**：
   - 如果实现多语言，确保所有文本都已翻译
   - 日期时间格式根据语言调整
