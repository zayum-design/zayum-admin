# Step 07 - 用户管理模块

## 目标
实现普通用户的完整CRUD操作，包括用户列表、新增、编辑、删除、状态管理等功能

## 后端实现

### 7.1 用户模块搭建
- 创建 `user` 模块（controller、service、module）
- 创建用户相关 DTO（CreateUserDto、UpdateUserDto、QueryUserDto）
- 配置模块依赖

### 7.2 DTO 定义

#### QueryUserDto（查询参数）
**字段**：
- `page`：页码，默认1
- `pageSize`：每页数量，默认10
- `username`：用户名（模糊搜索）
- `nickname`：昵称（模糊搜索）
- `email`：邮箱（模糊搜索）
- `mobile`：手机号（模糊搜索）
- `group_id`：用户组ID
- `gender`：性别筛选
- `status`：状态筛选（normal/hidden/locked）
- `created_at_start`：创建时间开始
- `created_at_end`：创建时间结束

#### CreateUserDto（创建用户）
**字段**：
- `username`：用户名（必填，唯一，2-20字符）
- `nickname`：昵称（必填，2-50字符）
- `password`：密码（必填，6-20字符）
- `email`：邮箱（必填，邮箱格式，唯一）
- `mobile`：手机号（必填，11位数字，唯一）
- `group_id`：用户组ID（必填）
- `avatar`：头像URL（可选）
- `gender`：性别（male/female/unknown，默认unknown）
- `birthday`：生日（可选）
- `status`：状态（默认 normal）

**验证规则**：
- 使用 class-validator 装饰器
- username、email、mobile 唯一性验证
- 密码强度验证

#### UpdateUserDto（更新用户）
**字段**：
- 与 CreateUserDto 类似，都是可选字段
- 密码可选（不填则不修改）

### 7.3 用户列表接口
**接口**：`GET /api/users`

**权限**：`user:list:view`

**查询参数**：参考 QueryUserDto

**业务逻辑**：
1. 构建查询条件（模糊搜索、精确匹配、日期范围）
2. 联表查询用户组信息（可选）
3. 分页查询
4. 排序（默认按创建时间倒序）
5. 隐藏敏感字段（password）
6. 返回列表数据和分页信息

**响应格式**：
```
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": 1,
        "username": "testuser",
        "nickname": "测试用户",
        "email": "test@example.com",
        "mobile": "13800138001",
        "avatar": null,
        "gender": "male",
        "birthday": "1990-01-01",
        "group_id": 1,
        "group_name": "普通用户组",
        "status": "normal",
        "created_at": "2024-01-01 00:00:00",
        "updated_at": "2024-01-01 00:00:00"
      }
    ],
    "pagination": {
      "page": 1,
      "pageSize": 10,
      "total": 100,
      "totalPages": 10
    }
  }
}
```

### 7.4 用户详情接口
**接口**：`GET /api/users/:id`

**权限**：`user:list:view`

**业务逻辑**：
1. 根据 ID 查询用户
2. 联表查询用户组信息
3. 隐藏密码字段
4. 用户不存在时返回 404

### 7.5 创建用户接口
**接口**：`POST /api/users`

**权限**：`user:list:create`

**请求参数**：参考 CreateUserDto

**业务逻辑**：
1. DTO 验证
2. 检查 username 是否已存在
3. 检查 email 是否已存在
4. 检查 mobile 是否已存在
5. 验证用户组是否存在
6. 密码使用 bcrypt 加密（轮数10）
7. 创建用户记录
8. 记录操作日志
9. 返回创建成功的用户信息（不含密码）

**错误处理**：
- 用户名已存在：400
- 邮箱已存在：400
- 手机号已存在：400
- 用户组不存在：400

### 7.6 更新用户接口
**接口**：`PUT /api/users/:id`

**权限**：`user:list:edit`

**请求参数**：参考 UpdateUserDto

**业务逻辑**：
1. DTO 验证
2. 检查用户是否存在
3. 如果修改 email，检查新邮箱是否被占用
4. 如果修改 mobile，检查新手机号是否被占用
5. 如果修改 password，使用 bcrypt 加密新密码
6. 如果修改 group_id，验证新用户组是否存在
7. 更新用户记录
8. 记录操作日志
9. 返回更新后的用户信息

**特殊处理**：
- 密码字段为空时不更新密码

### 7.7 删除用户接口
**接口**：`DELETE /api/users/:id`

**权限**：`user:list:delete`

**业务逻辑**：
1. 检查用户是否存在
2. 检查是否有关联数据（订单、评论等，根据业务需求）
3. 执行删除（软删除或硬删除）
4. 记录操作日志
5. 返回成功消息

**软删除方案**：
- 添加 `deleted_at` 字段
- 删除时设置 deleted_at = 当前时间
- 查询时自动过滤已删除记录

### 7.8 批量删除接口
**接口**：`DELETE /api/users/batch`

**权限**：`user:list:delete`

**请求参数**：
- `ids`：用户ID数组

**业务逻辑**：
- 批量验证和删除
- 使用事务处理

### 7.9 修改用户状态接口
**接口**：`PATCH /api/users/:id/status`

**权限**：`user:list:edit`

**请求参数**：
- `status`：新状态（normal/hidden/locked）

**业务逻辑**：
1. 验证状态值有效性
2. 更新状态
3. 如果设置为 locked，可考虑强制登出该用户
4. 记录操作日志

### 7.10 重置用户密码接口
**接口**：`POST /api/users/:id/reset-password`

**权限**：`user:list:edit`

**请求参数**：
- `new_password`：新密码（可选，不填则生成随机密码）

**业务逻辑**：
1. 生成或接收新密码
2. 验证密码强度
3. 使用 bcrypt 加密
4. 更新密码
5. 记录操作日志
6. 返回新密码（如果是随机生成的）

### 7.11 用户导出接口（可选）
**接口**：`GET /api/users/export`

**权限**：`user:list:export`

**业务逻辑**：
1. 根据查询条件获取用户数据
2. 生成 Excel 文件
3. 返回文件下载链接或直接返回文件流

### 7.12 用户导入接口（可选）
**接口**：`POST /api/users/import`

**权限**：`user:list:import`

**请求参数**：
- `file`：Excel 文件

**业务逻辑**：
1. 解析 Excel 文件
2. 验证数据格式
3. 批量创建用户
4. 返回导入结果（成功数、失败数、错误详情）

### 7.13 操作日志记录
- 在所有写操作后记录操作日志
- 记录内容：操作人、操作类型、操作对象、详情、时间、IP等

**表格列**：

### 8.4 新增用户
**触发**：点击"新增用户"按钮

**实现方式**：抽屉或对话框

**表单字段**：
- 用户名（必填，Input）
- 昵称（必填，Input）
- 密码（必填，Password，显示强度指示器）
- 确认密码（必填）
- 邮箱（必填，Input）
- 手机号（必填，Input）
- 用户组（必填，Select）
- 性别（Radio：男/女/未知）
- 生日（DatePicker）
- 头像（Upload）
- 状态（Radio，默认 normal）

**表单验证**：
- 用户名：2-20字符
- 密码：6-20字符，包含字母和数字
- 确认密码：与密码一致
- 邮箱：邮箱格式
- 手机号：11位数字

**提交流程**：
1. 表单验证
2. 调用创建接口
3. 成功后关闭表单并刷新列表

### 8.5 编辑用户
**触发**：点击表格行的"编辑"按钮

**实现方式**：抽屉或对话框

**表单字段**：

## 注意事项

1. **数据安全**：
   - 密码必须加密存储
   - 密码字段永不返回给前端
   - 敏感操作记录日志

2. **性能优化**：
   - 列表查询使用索引
   - 批量操作使用事务
   - 导出大量数据时考虑异步处理

3. **用户体验**：
   - 操作需要确认提示
   - 错误信息友好
   - 表单验证实时反馈

4. **业务规则**：
   - 用户名、邮箱、手机号唯一性
   - 用户组的关联验证
   - 账号状态的合理切换
