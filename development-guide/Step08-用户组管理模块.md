# Step 08 - 用户组管理模块

## 目标
实现用户组的CRUD操作和权限分配功能，完成用户端的RBAC权限系统

## 后端实现

### 8.1 用户组模块搭建
- 创建 `user-group` 模块（controller、service、module）
- 创建相关 DTO（CreateUserGroupDto、UpdateUserGroupDto、QueryUserGroupDto、AssignPermissionsDto）

### 8.2 DTO 定义

#### QueryUserGroupDto（查询参数）
**字段**：
- `page`：页码，默认1
- `pageSize`：每页数量，默认10
- `name`：组名（模糊搜索）
- `status`：状态筛选

#### CreateUserGroupDto（创建用户组）
**字段**：
- `name`：组名（必填，唯一，2-50字符）
- `description`：描述（可选，最多200字符）
- `status`：状态（默认 normal）

**验证规则**：
- name 必填且唯一
- description 长度限制

#### UpdateUserGroupDto（更新用户组）
**字段**：
- 与 CreateUserGroupDto 类似，都是可选字段

#### AssignPermissionsDto（分配权限）
**字段**：
- `permission_ids`：权限ID数组（必填）

**验证规则**：
- 数组不能为空
- 权限ID必须存在

### 8.3 用户组列表接口
**接口**：`GET /api/user-groups`

**权限**：`user:group:view`

**查询参数**：参考 QueryUserGroupDto

**业务逻辑**：
1. 构建查询条件
2. 分页查询
3. 统计每个组的用户数量
4. 返回列表数据和分页信息

**响应格式**：
```
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": 1,
        "name": "普通用户组",
        "description": "系统默认用户组",
        "status": "normal",
        "user_count": 100,
        "created_at": "2024-01-01 00:00:00",
        "updated_at": "2024-01-01 00:00:00"
      }
    ],
    "pagination": {
      "page": 1,
      "pageSize": 10,
      "total": 5,
      "totalPages": 1
    }
  }
}
```

### 8.4 用户组详情接口
**接口**：`GET /api/user-groups/:id`

**权限**：`user:group:view`

**业务逻辑**：
1. 根据 ID 查询用户组
2. 统计该组的用户数量
3. 查询该组拥有的权限列表
4. 用户组不存在时返回 404

**响应格式**：
```
{
  "code": 200,
  "data": {
    "id": 1,
    "name": "普通用户组",
    "description": "系统默认用户组",
    "status": "normal",
    "user_count": 100,
    "permissions": [
      {
        "id": 10,
        "name": "个人中心",
        "code": "profile"
      }
    ],
    "created_at": "2024-01-01 00:00:00",
    "updated_at": "2024-01-01 00:00:00"
  }
}
```

### 8.5 创建用户组接口
**接口**：`POST /api/user-groups`

**权限**：`user:group:create`

**请求参数**：参考 CreateUserGroupDto

**业务逻辑**：
1. DTO 验证
2. 检查组名是否已存在
3. 创建用户组记录
4. 记录操作日志
5. 返回创建成功的用户组信息

**错误处理**：
- 组名已存在：400

### 8.6 更新用户组接口
**接口**：`PUT /api/user-groups/:id`

**权限**：`user:group:edit`

**请求参数**：参考 UpdateUserGroupDto

**业务逻辑**：
1. DTO 验证
2. 检查用户组是否存在
3. 如果修改组名，检查新组名是否被占用
4. 更新用户组记录
5. 记录操作日志
6. 返回更新后的用户组信息

### 8.7 删除用户组接口
**接口**：`DELETE /api/user-groups/:id`

**权限**：`user:group:delete`

**业务逻辑**：
1. 检查用户组是否存在
2. 不允许删除默认用户组（ID=1，可选策略）
3. 检查该组下是否有用户
   - 如果有，提示需要先移除或转移用户
   - 或提供强制删除选项（将用户移至默认组）
4. 删除该组的权限关联（sys_role_permission 表）
5. 删除用户组记录
6. 记录操作日志
7. 返回成功消息

### 8.8 获取用户组权限接口
**接口**：`GET /api/user-groups/:id/permissions`

**权限**：`user:group:view`

**业务逻辑**：
1. 查询 sys_role_permission 表
2. 获取该用户组的所有权限ID
3. 查询权限详细信息
4. 构建权限树（可选）
5. 返回权限列表

**响应格式**：
```
{
  "code": 200,
  "data": {
    "permission_ids": [10, 11, 12],
    "permissions": [
      {
        "id": 10,
        "name": "个人中心",
        "code": "profile",
        "type": "menu"
      }
    ]
  }
}
```

### 8.9 分配权限接口
**接口**：`POST /api/user-groups/:id/permissions`

**权限**：`user:group:edit`

**请求参数**：参考 AssignPermissionsDto

**业务逻辑**：
1. 验证用户组是否存在
2. 验证所有权限ID是否存在
3. 使用事务处理：
   - 删除该组原有的所有权限关联（role_type='user_group'）
   - 批量插入新的权限关联
   - 提交事务
4. 记录操作日志
5. 清除该组所有用户的权限缓存（如有）
6. 返回成功消息

**响应格式**：
```
{
  "code": 200,
  "message": "权限分配成功"
}
```

### 8.10 获取用户组下的用户列表
**接口**：`GET /api/user-groups/:id/users`

**权限**：`user:group:view`

**业务逻辑**：
1. 查询该用户组下的所有用户
2. 支持分页
3. 返回用户列表

### 8.11 批量操作接口（可选）
- 批量删除用户组
- 批量修改状态

### 8.12 权限数据初始化
**用户端常见权限示例**：
- 个人中心（profile）
  - 查看个人信息（profile:view）
  - 编辑个人信息（profile:edit）
  - 修改密码（profile:change-password）
- 消息通知（notification）
  - 查看消息（notification:view）
  - 标记已读（notification:read）
  - 删除消息（notification:delete）

**注意**：
- 用户端权限通常不涉及系统管理功能
- 用户端权限主要控制用户能访问的功能模块
- 可以根据实际业务需求定义权限

## 前端实现

### 9.1 用户组管理页面
**路由**：`/user/group`

**页面结构**：
- 搜索区域（组名搜索、状态筛选）
- 操作按钮区（新增用户组）
- 数据表格
- 分页器

### 9.2 数据表格
**表格列**：
- 序号
- 组名
- 描述
- 用户数量（可点击查看用户列表）
- 状态（Tag 组件）
- 创建时间
- 更新时间
- 操作（按钮组）

**操作按钮**：
- 权限设置（权限：user:group:edit）
- 编辑（权限：user:group:edit）
- 删除（权限：user:group:delete）

### 9.3 新增用户组
**触发**：点击"新增用户组"按钮

**实现方式**：对话框或抽屉

**表单字段**：
- 组名（必填，Input）
- 描述（可选，TextArea）
- 状态（Radio，默认 normal）

**提交流程**：
1. 表单验证
2. 调用创建接口
3. 成功后关闭表单并刷新列表

### 9.4 编辑用户组
**触发**：点击表格行的"编辑"按钮

**实现方式**：对话框或抽屉

**表单字段**：
- 与新增类似，预填充当前用户组数据

### 9.5 删除用户组
**触发**：点击表格行的"删除"按钮

**实现方式**：确认对话框

**确认提示**：
```
确定要删除用户组 "xxx" 吗？该组下还有 N 个用户。
```

**提交流程**：
1. 检查该组是否有用户
2. 如果有用户，提示用户先移除用户
3. 确认后调用删除接口

### 9.6 权限设置功能
**触发**：点击表格行的"权限设置"按钮

**实现方式**：抽屉（Drawer，宽度较大）

**页面布局**：
- 左侧：权限树（Tree 组件）
- 右侧：用户组信息摘要

**权限树功能**：
- 显示所有权限的树形结构
- 支持勾选/取消勾选（Checkbox）
- 支持全选/反选
- 支持展开/折叠所有节点
- 支持按类型筛选
- 支持搜索权限

**权限树实现**：
- 使用 Antd Tree 组件
- checkable 属性开启复选框
- checkedKeys 绑定已选权限ID
- 父子节点关联

**数据加载**：
1. 打开抽屉时调用两个接口：
   - 获取所有权限树
   - 获取该用户组已有的权限ID列表
2. 设置 Tree 的 checkedKeys

**保存流程**：
1. 获取当前选中的所有权限ID
2. 调用分配权限接口
3. 成功后提示并关闭抽屉
4. 刷新列表（可选）

**权限范围说明**：
- 用户组的权限树可能与管理员组不同
- 只显示用户端相关的权限
- 不显示系统管理类权限

### 9.7 查看用户列表
**触发**：点击用户数量

**实现方式**：对话框或抽屉

**内容**：
- 显示该用户组下所有用户的表格

### 9.9 API Service

建议拆分组件：
- `UserGroupList.tsx`：主页面
- `UserGroupTable.tsx`：数据表格
- `UserGroupForm.tsx`：新增/编辑表单

## 注意事项

1. **权限范围**：
   - 用户组的权限应该是用户端权限的子集
   - 不应该包含系统管理类权限
   - 权限设计要符合业务需求

2. **权限更新时效性**：
   - 分配权限后，考虑是否需要强制用户重新登录
   - 或实现权限实时刷新机制

3. **默认用户组保护**：
   - 考虑是否需要一个不可删除的默认用户组
   - 新注册用户自动加入默认用户组

4. **级联删除**：
   - 明确删除用户组时，该组下用户的处理策略
   - 是否允许强制删除并转移用户

5. **权限继承**：
   - 当前设计是直接授权，不涉及继承
   - 如需支持继承，需调整数据结构和逻辑

6. **与管理员组的区分**：
   - 管理员组和用户组使用同一套权限表
   - 通过 role_type 字段区分
   - 但权限范围和功能可能不同
